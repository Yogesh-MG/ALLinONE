# Contributing to MediTrackPros

Thank you for your interest in contributing to **MediTrackPros**. This project aims to provide an open, extensible, and practical hospital workflow management system.

---

## 🧭 Ways to Contribute

You can contribute through:

* Bug fixes
* Feature development
* Documentation improvements
* Performance optimization
* Security enhancements
* Test coverage

---

## 🛠 Development Setup

1. Fork the repository
2. Clone your fork

```bash
git clone https://github.com/<your-username>/Meditrackpro.git
cd Meditrackpro
```

3. Create a feature branch

```bash
git checkout -b feature/your-feature-name
```

---

## 📐 Code Guidelines

* Follow Django REST Framework best practices
* Keep modules domain-specific
* Enforce hospital-level data isolation
* Do not hardcode secrets
* Use serializers for all API I/O

---

## ✅ Pull Request Checklist

Before submitting a PR:

* Code builds successfully
* No sensitive data committed
* Changes are documented
* APIs remain backward compatible (unless discussed)

---

## 📬 Communication

For major changes, please open an issue before starting development.

Thank you for contributing to MediTrackPros.
